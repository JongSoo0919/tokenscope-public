# Prompt Coach Wiki

이 위키는 제품/프로젝트 지식이 아니라 AI에게 일을 맡기는 방법을 다룹니다.
TokenScope가 감지한 세션 패턴과 사용자 질문을 이 지식과 함께 RAG로 조회해 다음 요청을 더 작고 명확하게 바꾸는 데 사용합니다.

## 사용 원칙

- 질문을 고치기 전에 먼저 사용자의 의도를 한 문장으로 요약한다.
- 토큰 절약은 무조건 짧게 쓰기가 아니라, 불필요한 탐색과 반복을 줄이는 방향으로 판단한다.
- 코딩 에이전트 요청은 목표, 범위, 금지 사항, 완료 조건을 분리한다.
- 긴 지침은 항상 넣지 말고 필요할 때만 참조한다.
- 실패가 반복되면 같은 요청을 다시 보내지 말고 에러, 시도한 방법, 원하는 판단을 명시한다.

## 주요 문서

- `prompt-patterns.md`: 좋은 질문 구조와 나쁜 습관
- `token-economy.md`: 토큰과 캐시 절약 원칙
- `coding-agent-prompts.md`: 코딩 에이전트용 요청 템플릿
- `rewrite-rules.md`: TokenScope에서 질문을 재작성할 때의 규칙
