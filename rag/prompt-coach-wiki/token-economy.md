# 토큰 절약과 컨텍스트 관리

## 근거 요약

Microsoft의 Azure OpenAI 문서는 프롬프트 캐싱이 동일한 앞부분을 가진 긴 프롬프트의 지연 시간과 비용을 줄인다고 설명한다. 캐시 효과를 높이려면 반복되는 내용이 메시지 앞쪽에 안정적으로 위치해야 한다.

Anthropic의 컨텍스트 엔지니어링 글은 컨텍스트가 유한한 자원이며, 길어질수록 모델의 주의가 분산될 수 있다고 설명한다. 최소 프롬프트는 단순히 짧은 프롬프트가 아니라, 필요한 정보를 충분히 주되 불필요한 토큰을 줄인 프롬프트다.

출처:
- Microsoft Learn, Prompt caching: https://learn.microsoft.com/en-us/azure/foundry/openai/how-to/prompt-caching
- Anthropic Engineering, Effective context engineering for AI agents: https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents

## 토큰을 줄이는 진짜 방법

토큰 절약은 질문을 무조건 짧게 쓰는 일이 아니다. 아래 낭비를 줄이는 것이 더 중요하다.

- 관련 없는 파일 탐색
- 같은 에러 반복
- 한 세션에서 기획, 구현, 검증이 섞이는 일
- 항상 읽히는 전역 지침의 비대화
- 긴 로그 전체 붙여넣기
- 이전 대화 전체를 다시 설명하기

## 캐시 친화적인 요청

캐시가 있는 모델이나 도구에서는 안정적인 앞부분이 중요하다.

좋은 구조:

```text
역할: 너는 코드 리뷰어다.
고정 규칙: 버그, 회귀 위험, 누락 테스트만 본다.
출력 형식: 심각도순 목록.

이번 입력:
파일: ...
변경 요약: ...
질문: ...
```

나쁜 구조:

```text
이번엔 이 파일 보고, 아 맞다 규칙은 이렇고, 저번에는 이랬고, 전체적으로...
```

## 긴 입력을 줄이는 방법

- 로그 전체 대신 에러 마지막 30줄, 실행 명령, 기대 결과를 준다.
- 이전 세션 전체 대신 결정 사항, 변경 파일, 남은 작업, 검증 명령만 넘긴다.
- 코드 전체 대신 관련 함수/파일 경로와 증상을 준다.
- 불확실한 경우 "먼저 필요한 파일만 찾아서 계획을 말해줘"라고 요청한다.

## 세션 분리 기준

아래 중 하나가 발생하면 새 세션이 낫다.

- 기획에서 구현으로 단계가 바뀐다.
- 구현에서 검증/QA로 단계가 바뀐다.
- 프로젝트나 저장소가 바뀐다.
- 실패 원인을 새로 조사해야 한다.
- 이전 대화의 80%가 현재 목표와 무관하다.
