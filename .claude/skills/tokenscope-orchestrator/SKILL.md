# TokenScope Orchestrator

## 개요
TokenScope 하네스의 전체 워크플로우를 조율합니다. 토큰 낭비 진단, 처방 생성, UI 구현, 대시보드, 수정 미리보기 기능을 포함합니다.

## 실행 모드
**에이전트 팀** — 4명의 전문가가 협업하여 실시간 조율 및 피드백 교환

## Phase 0: 컨텍스트 확인

워크플로우 시작 시 기존 산출물 존재 여부를 확인하여 실행 모드를 결정합니다:
- `_workspace/` 존재 + 사용자가 부분 수정 요청 → **부분 재실행** (해당 에이전트만 재호출)
- `_workspace/` 존재 + 사용자가 새 입력 제공 → **새 실행** (기존 _workspace를 `_workspace_prev/`로 이동)
- `_workspace/` 미존재 → **초기 실행**

## Phase 1: 데이터 수집

**실행 모드:** 에이전트 팀

### 작업
1. `parser-agent`가 JSONL 파일을 파싱하여 세션 데이터 구조화
2. `analyzer-agent`가 CLAUDE.md를 읽고 섹션별 토큰 계산
3. 모든 세션의 메타데이터 생성 (총 토큰, 비용 추정, 날짜)

### 데이터 전달
- 파일 기반: `_workspace/01_parsed_sessions.json`
- 태스크 기반: `TaskCreate`로 작업 할당

### 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| 경로 없음 | "Claude Code가 설치되어 있나요?" 메시지 |
| 권한 부족 | "파일 접근 권한이 없습니다." 메시지 |
| 파일 잠김 | 해당 파일 skip + "진행 중인 세션은 종료 후 분석됩니다" 메시지 |
| 포맷 불일치 | "로그 포맷이 변경된 것 같습니다." 메시지 |

## Phase 2: 패턴 분석

**실행 모드:** 에이전트 팀

### 작업
1. `analyzer-agent`가 세션 데이터를 분석하여 패턴 감지
2. 건강 점수 계산 (캐시 효율, 도구 성공률, 컨텍스트 밀도, CLAUDE.md 건강도, 반복 요청 건강도)
3. 대시보드 집계 (전체 평균 건강 점수, 자주 하는 실수)

### 데이터 전달
- 파일 기반: `_workspace/02_analyzed_sessions.json`
- 태스크 기반: `TaskCreate`로 작업 할당

### 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| CLAUDE.md 없음 | CONTEXT_BLOAT 패턴 skip |
| 메시지 없음 | 빈 결과 반환 |
| 계산 오류 | 0점 반환 + 경고 로그 |

## Phase 3: 처방 생성

**실행 모드:** 에이전트 팀

### 작업
1. `prescriber-agent`가 패턴별 처방 생성
2. CLAUDE.md diff 생성
3. 질문 가이드 작성
4. 수정 전/후 토큰 비교

### 데이터 전달
- 파일 기반: `_workspace/03_prescriptions.json`
- 태스크 기반: `TaskCreate`로 작업 할당

### 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| CLAUDE.md 없음 | info-only 처방 반환 |
| diff 생성 실패 | 원본/수정본 텍스트만 반환 |
| 백업 실패 | 수정 중단 + 에러 메시지 |

## Phase 4: UI 구현

**실행 모드:** 에이전트 팀

### 작업
1. `ui-agent`가 React 컴포넌트 개발
2. 건강 점수 상세 탭 구현
3. 건강 점수별 필터링 구현
4. 수정 미리보기 기능 구현
5. 대시보드 구현
6. 질문 가이드 컴포넌트 구현

### 데이터 전달
- 파일 기반: `_workspace/04_ui_components.tsx`
- 태스크 기반: `TaskCreate`로 작업 할당

### 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| 데이터 없음 | "데이터를 불러오는 중..." 메시지 |
| 로딩 중 | 스피너 표시 |
| 에러 발생 | 친절한 에러 메시지 + 재시도 버튼 |

## Phase 5: 통합 및 테스트

**실행 모드:** 에이전트 팀

### 작업
1. 모든 컴포넌트 통합
2. 기능 테스트
3. 사용자 경험 테스트
4. 버그 수정

### 데이터 전달
- 파일 기반: `_workspace/05_integrated_app.tsx`
- 태스크 기반: `TaskCreate`로 작업 할당

### 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| 통합 실패 | 해당 컴포넌트만 재작업 |
| 테스트 실패 | 버그 수정 후 재테스트 |

## 팀 통신 프로토콜

### 메시지 수신 대상
- **parser-agent**: `analyzer-agent`, `prescriber-agent`
- **analyzer-agent**: `prescriber-agent`, `ui-agent`
- **prescriber-agent**: `ui-agent`
- **ui-agent**: 모든 에이전트

### 작업 요청 범위
- **parser-agent**: JSONL 파싱, 캐시 관리
- **analyzer-agent**: 패턴 감지, 건강 점수 계산, 대시보드 집계
- **prescriber-agent**: 처방 생성, diff 생성, 질문 가이드, 수정 전/후 비교
- **ui-agent**: 컴포넌트 개발, UX 설계

## 에러 핸들링

### 1회 재시도 후 재실패 시 해당 결과 없이 진행
- 보고서에 누락 명시
- 상충 데이터는 삭제하지 않고 출처 병기

### 에러 유형별 전략

| 에러 유형 | 처리 방법 |
|----------|----------|
| 데이터 없음 | 해당 Phase skip + 보고서에 누락 명시 |
| 파싱 실패 | 해당 세션 skip + 경고 로그 |
| 계산 오류 | 0점 반환 + 경고 로그 |
| UI 렌더링 실패 | 에러 메시지 표시 + 재시도 버튼 |

## 팀 크기
- **팀원 수**: 4명 (parser-agent, analyzer-agent, prescriber-agent, ui-agent)
- **팀원당 작업 수**: 4-5개

## 테스트 시나리오

### 정상 흐름
1. 앱 시작 → 세션 목록 표시
2. 세션 선택 → 진단 결과 표시
3. "상세" 탭 클릭 → 건강 점수 상세 표시
4. "미리보기" 클릭 → 수정 전/후 토큰 비교 표시
5. "적용" 클릭 → CLAUDE.md 수정 + 백업
6. 대시보드 → 전체 평균 건강 점수 + 자주 하는 실수 표시

### 에러 흐름
1. CLAUDE.md 없음 → CONTEXT_BLOAT 패턴 skip
2. JSONL 파싱 실패 → 해당 세션 skip + 경고 로그
3. 백업 실패 → 수정 중단 + 에러 메시지
4. UI 렌더링 실패 → 에러 메시지 표시 + 재시도 버튼

## 후속 작업 지원

### 오케스트레이터 description에 후속 키워드 포함
- "다시 실행", "재실행", "업데이트", "수정", "보완"
- "{도메인}의 {부분작업}만 다시"
- "이전 결과 기반으로", "결과 개선"

### 오케스트레이터 Phase 1에 컨텍스트 확인 단계 추가
- `_workspace/` 존재 + 사용자가 부분 수정 요청 → **부분 재실행**
- `_workspace/` 존재 + 사용자가 새 입력 제공 → **새 실행**
- `_workspace/` 미존재 → **초기 실행**

### 에이전트 정의에 재호출 지침 포함
- 이전 결과 파일이 존재하면 읽고 개선점을 반영
- 사용자 피드백이 주어지면 해당 부분만 수정
