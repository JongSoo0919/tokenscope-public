# Analyzer Skill

## 개요
세션 데이터를 분석하여 토큰 낭비 패턴을 감지하고 건강 점수를 계산합니다.

## 작업 원칙
1. **규칙 기반 분석** — LLM 없이 정확한 패턴 감지
2. **수치화된 진단** — 모든 판단을 숫자로 표현
3. **비개발자 친화적** — 기술 용어를 일반 언어로 변환

## 패턴 분류 기준

### CONTEXT_BLOAT
- **판별**: system 토큰 / 세션 총 토큰 > 30%
- **심각도**: HIGH (>4000토큰), MEDIUM (>2000토큰), LOW (>667토큰)
- **비개발자 설명**: "CLAUDE.md가 너무 길어서 매 요청마다 불필요한 토큰을 소모합니다."

### RETRY_STORM
- **판별**: 동일 user 메시지 3회 이상 반복
- **심각도**: HIGH (10+ 중복), MEDIUM (5+ 중복), LOW (3+ 중복)
- **비개발자 설명**: "같은 명령이 여러 번 반복되어 토큰을 낭비하고 있습니다."

### TOOL_THRASH
- **판별**: 동일 tool 연속 3회 이상 + 모두 error
- **심각도**: HIGH (6+ 실패), MEDIUM (4+ 실패), LOW (3+ 실패)
- **비개발자 설명**: "도구가 계속 실패하고 있어 토큰을 낭비하고 있습니다."

## 건강 점수 계산

### 점수 구성
- **캐시 효율 (30%)**: 캐시 적중률 (0-100)
- **도구 성공률 (25%)**: 도구 성공률 (0-100)
- **컨텍스트 밀도 (20%)**: 출력/입력 비율 (0-100)
- **CLAUDE.md 건강도 (15%)**: CLAUDE.md 무게 (0-100)
- **반복 요청 건강도 (10%)**: 반복 메시지 없음 (0-100)

### 비개발자용 설명
- **캐시 효율**: "캐시를 얼마나 잘 사용하고 있나요?"
- **도구 성공률**: "도구가 얼마나 잘 작동하고 있나요?"
- **컨텍스트 밀도**: "출력 대비 입력이 적절한가요?"
- **CLAUDE.md 건강도**: "CLAUDE.md가 너무 길지 않나요?"
- **반복 요청 건강도**: "같은 명령이 반복되고 있나요?"

## 대시보드 집계

### 전체 평균 건강 점수
- 최근 30일 세션의 건강 점수 평균
- 트렌드 그래프 (일별 평균)

### 자주 하는 실수
- 패턴별 발생 빈도
- 상위 5개 패턴 표시
- 비개발자용 설명 포함

## 에러 핸들링
| 에러 상황 | 처리 방법 |
|----------|----------|
| CLAUDE.md 없음 | CONTEXT_BLOAT 패턴 skip |
| 메시지 없음 | 빈 결과 반환 |
| 계산 오류 | 0점 반환 + 경고 로그 |
