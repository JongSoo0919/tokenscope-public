# 코딩 에이전트 요청 템플릿

## 구현 요청

```text
목표: <원하는 동작>을 구현해줘.
범위: <수정 가능한 파일/디렉터리>만 건드려.
금지: 전체 재작성, 무관한 리팩터링, 삭제는 하지 마.
진행: 변경 전 3줄 계획을 말하고, 구현 후 <검증 명령>을 실행해.
완료 조건: <사용자가 확인할 수 있는 결과>가 나오면 끝.
```

## 디버깅 요청

```text
바로 고치지 말고 먼저 원인을 좁혀줘.
현상: <무슨 일이 일어남>
재현: <명령/절차>
에러: <핵심 로그 30줄 이하>
기대 결과: <정상 동작>
요청: 가능한 원인 3개를 우선순위로 정하고, 1순위만 검증한 뒤 최소 수정해.
```

## 리뷰 요청

```text
코드 리뷰만 해줘.
대상: <파일/PR/커밋>
관점: 버그, 회귀 위험, 누락 테스트, 보안 문제만 봐.
제외: 스타일 취향, 네이밍, 대규모 리팩터링.
출력: 심각도순으로 파일/라인 근거와 함께 써줘.
```

## 검증 요청

```text
수정하지 말고 검증만 해줘.
대상: <기능/화면/API>
검증: <빌드, 테스트, curl, 브라우저 흐름>
출력: 성공/실패와 실패 원인만 요약해.
```

## 기획 요청

```text
아직 코드 수정하지 마.
아이디어: <만들고 싶은 것>
목표 사용자: <누가 쓰는지>
제약: <시간, 기술, 범위>
요청: MVP 범위, 리스크, 작업 목록을 10줄 이하로 정리해.
```
