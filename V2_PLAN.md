# [기획서] TokenScope v2: 지능형 처방 시스템 (Intelligent Prescriber)

## 1. 개요
현재 TokenScope(v1)는 정해진 규칙에 따라 토큰 낭비를 감지하고 단순한 텍스트 기반 조언을 제공합니다. v2에서는 **LLM(대규모 언어 모델)**을 도입하여, 에이전트의 실패 패턴을 실시간으로 분석하고 구체적인 **`SKILL.md` 수정 코드**를 생성하는 "자율 최적화 루프"를 완성합니다.

## 2. LLM 도입 전략 비교: 로컬(Ollama) vs API Key(OpenAI/Anthropic)

| 비교 항목 | **로컬 LLM (Ollama)** | **API Key (OpenAI/Anthropic)** |
| :--- | :--- | :--- |
| **보안/프라이버시** | **최상.** 데이터가 기기 밖으로 나가지 않음. | **중.** 외부 서버를 거침. |
| **비용** | **무료.** 추가 비용 없음. | **종량제.** 사용량에 따라 과금. |
| **지능(Reasoning)** | **중.** 복잡한 맥락 이해가 소폭 낮을 수 있음. | **최상.** 최첨단 모델로 정교한 분석 가능. |
| **인터넷 연결** | **오프라인 가능.** | **필수.** |
| **추천 대상** | 개인 개발자, 보안 중시 프로젝트. | 최고 수준의 분석 품질을 원하는 유저. |

**전략:** 기본적으로 **Ollama를 우선 지원**하되, **API Key 입력 옵션**을 제공하는 하이브리드 방식으로 설계.

## 3. 시장 분석 및 경쟁사 조사
- **ccusage (Open Source):** 통계 중심. "왜"와 "어떻게"가 부족함.
- **LangSmith (Enterprise):** 무겁고 설정이 복잡함.
- **TokenScope 차별점:** 로컬 워크플로우에 즉시 적용 가능한 경량 UI + 실제 코드 수정(diff) 제공.
- **멀티 프로바이더 확장:** Claude뿐만 아니라 Gemini(Google), Codex(OpenAI) 로그 포맷을 통합 지원하여 범용적인 에이전트 최적화 도구로 포지셔닝.

## 4. 핵심 기능 상세
- **멀티 프로바이더 로그 파싱:**
  - **Claude (Anthropic):** 기존 JSONL 구조 유지.
  - **Gemini (Google):** `input_tokens`, `output_tokens` 필드 매핑 및 `tool_use` 구조 대응.
  - **Codex (OpenAI):** 독자적인 메타데이터 구조 및 토큰 계산 방식 적용.
- **에러 원인 분석:** `TOOL_THRASH` 발생 구간의 로그를 분석하여 근본 원인 파악.
- **맞춤형 SKILL.md 생성:** 분석된 원인을 방지하기 위한 가드레일 지침 생성.
- **절약 시뮬레이션:** 처방 적용 시 예상되는 토큰 절약량 가시화.

## 5. 기대 효과 및 발전 방향
- **비용 절감:** 낭비 토큰 20~30% 제거.
- **생산성 향상:** 에이전트 무한 루프 대기 시간 제거.
- **로드맵:** v2.5 자동 자가 치유(Self-Healing), v3.0 벤치마킹 플랫폼 확장.
